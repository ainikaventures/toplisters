# Contributing to TopListers

Thank you for your interest in contributing! Here's everything you need to know.

## Before you start

By submitting a pull request, you agree to our Contributor License Agreement (CLA). This is required so TopListers can be offered under a dual license model. The CLA bot will prompt you automatically on your first PR.

## How to contribute

1. Fork the repository
2. Create a branch: `git checkout -b feature/your-feature-name`
3. Make your changes
4. Write or update tests where relevant
5. Run the linter: `ruff check backend/app/`
6. Submit a pull request with a clear description

## What we welcome

- Bug fixes
- New visualisation ideas for the public dashboard
- Performance improvements
- Translations / internationalisation
- Documentation improvements
- New country support for the heatmap

## What we won't accept

- Changes that weaken GDPR compliance or privacy protections
- Features that store PII in public sightings
- Dependencies with incompatible licenses

## Code style

- Python: follow PEP8, use ruff for linting
- React: functional components with hooks only
- Commits: use conventional commits (`feat:`, `fix:`, `docs:`, `chore:`)
